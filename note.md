# AIに話しかけたら3時間で「俺専用燃費アプリ」が爆誕した話。App Storeよ、さようなら。

## ── そのアプリ、本当に使いやすいですか？

正直に告白します。

ガソリンスタンドで給油するたびに、スマホを取り出して燃費アプリを開くんですが、毎回ちょっとしたストレスを感じていたんですよ。

広告がバーン。「プレミアム登録で広告非表示！」バーン。  
やっと入力画面にたどり着いたと思えば、項目が多い。メモ欄、タグ付け、写真添付……**いや、今ガソリン入ってるんだけど！**

給油口を開けて、ノズルを刺して、数字を確認して、その数十秒の間にサクッと入力したい。それだけなのに、アプリが邪魔をしてくる。

「これ、自分で作れるんじゃないか？」

その一言が、すべての始まりでした。

今回、**Claude Code** というAI開発ツールをはじめて本格的に使ってみました。

「ターミナルで動くAI」と聞いて最初はちょっと身構えたんですが、これが想像をはるかに超えていました。

やったことは、シンプルです。要件を日本語で書いたプロンプトを投げただけ。

```
新規に、ガソリン代と燃費をスマートに記録する自分専用のWebアプリを作成してください。
デザインは車のコックピット感に馴染む、ダークモード基調のスタイリッシュなUI。
スマホの片手操作で3秒入力完了できるように...
```

すると、**ものの数分でこんな画面が錬成されました。**

![ダークモードの燃費トラッカー画面]

……これ、自分で書いたんじゃないですよ？AIが出してきたコードです。

深いチャコール（`#11141a`）の背景に、デジタルブルーのアクセント。数字はメーター風のフォントサイズで大きく表示。まるで車のインフォテインメントシステムのような仕上がりに、正直テンションが爆上がりしました。

「AIってこんなことができるの？」という感覚、令和の個人開発の醍醐味だと思います。

## こだわりポイント①：サーバー不要・維持費ゼロ

まず設計として一番大事にしたのが、**「余計なものを一切持たない」** という哲学です。

よくある燃費アプリは、会員登録してクラウドに保存して……という構造になっていますが、ぶっちゃけ自分一人しか使わないツールにそこまでいりません。

このアプリはすべてのデータをブラウザの **LocalStorage** に保存します。サーバーなし。データベースなし。月額費用ゼロ円。

HTMLファイルが一枚あればどこでも動く。シンプル・イズ・ベスト。

## こだわりポイント②：スタンドで3秒で入力できるカスタムキーパッド

ここが今回一番のこだわりポイントです。

市販のアプリが「入力しにくい」最大の原因のひとつは、**スマホの標準キーボード** にあると思っています。数字を入力するたびに巨大なキーボードがせり上がってきて、画面が半分見えなくなる。しかも「完了」ボタンが押しにくい位置にある。

だからこのアプリでは、**完全にオリジナルのカスタムナンバーパッド** を実装しました。

```
[ 1 ][ 2 ][ 3 ]
[ 4 ][ 5 ][ 6 ]  [⌫]
[ 7 ][ 8 ][ 9 ]
[ · ][ 0 ]      [次へ →]
```

タップしたらこのキーパッドがスライドアップして、OS標準のキーボードは一切出てこない。ボタンは大きく、指が太くても絶対に押せる設計。「次へ →」を押すと自動的に次の入力欄に進むので、**Odoメーター → 給油量 → 金額** の3フィールドを流れるように入力できます。

これを愛車に取り付けたマグネット式のスマホホルダーに固定して使うと、もう最高です。

> 📱 **[ここに車載スマホホルダーのアフィリエイトリンク]**
> （ガソリンスタンドで片手操作するなら、マグネット式でワンタッチで固定・取り外しができるタイプが断然おすすめです）

## こだわりポイント③：入力しながら「リアルタイム」で燃費が見える

これ、作ってみて「天才では？」と自画自賛したポイントです（笑）。

普通の燃費アプリって、「記録する」ボタンを押して初めて燃費が表示されますよね。でもこのアプリは違います。

Odoメーターの数字を入力した瞬間に、画面下のプレビューエリアに「**前回からの走行距離: 420 km**」と即座に表示されます。続けて給油量を入力すると「**推定燃費: 11.8 km/L**」がリアルタイムで更新される。

これ、何がいいかというと**タイプミスを即座に察知できる**んですよ。「15420km」と入れるところを「1542km」と間違えたら、走行距離が42kmとかおかしな数字になるので、記録ボタンを押す前に気づけます。

給油量の欄を入力しながら、自分の車の燃費がリアルタイムで変化していくのを眺めるのが、謎に楽しい。

## こだわりポイント④：満タン法 vs 部分給油、ちゃんと対応している

燃費の計算方法って、実は少し奥が深いんです。

基本は「**満タン法**」── 満タンにしてから次も満タンにするまでの走行距離 ÷ 給油量 で計算します。これが最も正確な実燃費の計測法です。

でも途中で「あ、ちょっとだけ入れておこう」という**部分給油**をすることもありますよね。このとき、部分給油分を無視して計算すると誤差が出てしまいます。

このアプリでは入力時に「満タン / 部分給油」をトグルで切り替えられて、**部分給油したリッター数は次の満タン給油まで内部で積み上げ、正確な合計給油量で燃費を計算**します。ここまで考えて作られている燃費アプリ、市販品でも少ないと思いますよ。

## こだわりポイント⑤：燃費が数値で見えると、沼にハマる

これが予想外の副産物でした。

燃費を記録し始めると、グラフが育っていくんです。「先月より0.3km/L上がった」「冬になると確かに燃費が落ちる」「高速道路を使った週は明らかに数値がいい」……

数値で見えると、**燃費改善が趣味になる**んですよ。

タイヤの空気圧を適正値に戻したら0.5km/L改善した、とか。アイドリングを気にするようになった、とか。

さらに燃費改善に効果があると言われているのが、**燃料添加剤**の定期投入です。エンジン内部の汚れを落として燃焼効率を上げてくれるやつです。実際に私も試してみたところ、数字として変化が見えてなかなか面白かったです。

> ⛽ **[ここにガソリン添加剤（フューエルワン等）のアフィリエイトリンク]**
> （3〜4回給油に1本のペースで入れるだけ。燃費トラッカーで数値の変化を追うとより実感しやすいです）

## こだわりポイント⑥：円/km という新しい指標

このアプリで個人的に気に入っているのが、「**円/km**」という表示です。

「実燃費 12.3 km/L」と言われても、「じゃあ1km走るのにいくらかかってるの？」って直感的にわかりにくいですよね。

「円/km」はそれをズバリ表示します。ガソリンが170円/Lで燃費が12.3km/Lなら、1km走るのに約13.8円。これが数値で出ると、「100kmのドライブで1,380円か」とか、「この週は渋滞が多くて円/kmが悪化してる」とかが直感的に把握できます。

ガソリン代の節約を考えるなら、給油ごとの単価を下げることも重要です。今、石油会社系のクレジットカードは「リッターあたり数円引き」が当たり前になっていて、毎回満タン（約50L）入れる人なら年間でかなりの金額差になります。

> 💳 **[ここに石油会社系おすすめクレジットカードのアフィリエイトリンク]**
> （燃費をアプリで管理しながら、給油単価もカードで削減。この二刀流が最強の防衛策です）

## iPhoneのホーム画面に「アイコン」として入れている

このアプリ、実はPWA（Progressive Web App）にもなっています。

SafariでアプリのURLを開いて、「ホーム画面に追加」するだけで、ネイティブアプリと見分けがつかないアイコンがホーム画面に登場します。タップするとSafariのアドレスバーすら消えて、完全にアプリとして起動します。

App Storeへの申請不要。審査不要。もちろん無料。

自分で作ったアプリが自分のスマホのホーム画面に鎮座している感覚、これが個人開発の快感です。

## まとめ：「欲しいなら作れ」の時代が来た

このアプリの開発にかかった時間、実質3時間ほどです。

コーディングの経験がない人でも、Claude Codeにやりたいことを日本語で伝えれば、今日の午後から始めて夜には動くアプリができます。プロンプトの書き方さえ覚えれば、あとはAIが大部分を錬成してくれる。

「自分には無理」という時代は終わりました。

私が個人的に一番感じた価値は、**「自分が本当に使いたいと思える粒度でカスタマイズできる」** ことです。市販のアプリは100人が使えるように設計されているから、どうしても自分にとって不要な機能が入ってくる。でも自分で作れば、自分にとってのベストを追求できる。

ガソリンスタンドでスマホをサッと取り出して、3タップで入力完了して、燃費の数字を確認してニヤリとする。そんな小さなワクワクを、AIが作ってくれました。

次は何を作ろうかな、と今からすでに考えています。
